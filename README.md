# MVP: Проверка работ стажеров-аналитиков

Локальный CLI-инструмент для проверки задания `Аналитики -> 2026 -> Сбор требований`.

## Что умеет MVP

- Принимает один файл ответа стажера (`.docx` или `.pdf`)
- Автоматически использует внутренний пакет задания (кейс, интервью, шаблон, критерии)
- Делает rule-based проверку и, опционально, LLM-оценку
- Возвращает:
  - `JSON` для машинной обработки
  - `Markdown` отчет для человека
- Формирует раздельные блоки:
  - для наставника
  - для стажера
- Показывает риск-сигналы ИИ только как сигналы (не как доказанный факт)
- Работает без калибровочных примеров, но умеет их подключать позже

## Быстрый старт

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python run_check.py --input "C:\path\to\answer.docx"
```

Результаты по умолчанию сохраняются в папку `outputs/`.

## Веб-интерфейс

```powershell
python run_web.py
```

После запуска откройте:

```text
http://127.0.0.1:8000
```

Через интерфейс можно:

- загрузить `DOCX/PDF`,
- выбрать профиль,
- запустить проверку,
- открыть готовые `Markdown/JSON` отчеты.

## Аргументы

```text
python run_check.py --input <path> [--profile analysts_2026_requirements] [--output-dir outputs] [--disable-llm]
```

- `--input`: путь к ответу стажера (`.docx`/`.pdf`)
- `--profile`: идентификатор профиля задания (по умолчанию `analysts_2026_requirements`)
- `--output-dir`: куда сохранять результаты
- `--disable-llm`: отключить LLM-слой и оставить только rule-based анализ

## Опциональная LLM-проверка

По умолчанию LLM включен, но используется только если:

- установлен пакет `openai`
- задан `OPENAI_EDU_API_KEY` или `OPENAI_API_KEY`

Можно указать два ключа:

```powershell
$env:OPENAI_EDU_API_KEY="учебный ключ"
$env:OPENAI_API_KEY="коммерческий ключ"
```

Сначала используется учебный ключ `OPENAI_EDU_API_KEY`. Если он недоступен, закончилась квота или ключ отклонен API, проверка автоматически повторяется с коммерческим `OPENAI_API_KEY`.
Для коммерческого ключа можно оставить общий `OPENAI_BASE_URL`. Учебный ключ использует прямой OpenAI API, если отдельно не задан `OPENAI_EDU_BASE_URL`.

Дополнительно можно задать модель:

```powershell
$env:OPENAI_MODEL="gpt-4o-mini"
```

Если LLM недоступен или произошел сбой, сценарий не падает: отчет строится по rule-based части.
